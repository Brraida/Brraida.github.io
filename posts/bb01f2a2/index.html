自动登录校园网

    优化自动登录校园网的代码。

需求：

    学校不是通宵供电，会断电，这导致每天早上都需要进行路由器的账号登录。为解决这个问题，写一个shell脚本。

    完成自动登录这个目标，可以看出是两块，一块是自动，一块是登录。路由器刷的是老毛子固件，该固件有自己的后台管理界面，在管理界面加自动化就行（更一般的方法，可以考虑linux中的crontab）；登录这边将使用curl。

步骤：

首先是用edge直接获取curl（bash）的代码。   

  1. 按f12，在Network中勾选Preserve log，然后按下记录按钮。

  1. 输入账号密码，获得Get请求。

  2. 在上图红线处右键，选copy，在选择**copy as cURL(bash)**。

其次编写shell script，如下：

#!/bin/bash
# Program:
#            auto login school nerwork by writing shell scripts in router
# History:
# 2022/05/06        brraida        First release
#PATH=/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/bin:/usr/local/sbin
#export PATH

maxTry=3        # 尝试最大次数
currentTry=0    # 目前正在尝试的次数
httpUrl=''      # get请求的url
response=404    # 初始化最开始的响应为404

# 获取url的状态码
function getStatus(){
    response=$(curl -o /dev/null -s -w "%{http_code}\n" $httpUrl)
}

# 当目前的尝试次数为最大次数时，则停止尝试，并且在日志中打印失败信息
while [ $currentTry -ne $maxTry ]
do
    getStatus;
    # 200状态码为请求成功状态码，若为此，则成功跳出
    if [ $response -eq 200 ]; then
        logger "connection success"
        break
    else
        currentTry=$(($currentTry+1))
        logger "retry $currentTry"
    fi
done

if [ $currentTry -eq $maxTry ]; then
    logger "try max, fail"
fi

    其中httpUrl填写的内容为在第一步中获取的get请求的url，可以将其他curl参数抛弃（如-H）。

    日志文件在/tmp/syslog.log

    将该文件保存在/etc/storage/目录下

最后在后端界面加上该文件地址即可。

【自定义设置】–>【脚本】–>【在路由器启动后运行】在最后添加：

/etc/storage/autoLoginCompusNetwork

    其实以上shell脚本也可以直接写在这里，就是不是更适用的选择。
