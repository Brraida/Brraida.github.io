6月7号 :sunny:

继承(inheritance)

静态方法

​ @staticmethod

类方法

​ @classmethod

区别

​ 类方法的第一个参数必须是指向本身的引用，而静态方法可以没有任何参数。

受保护变量和方法(protected)

​ 单下划线

  1. 受保护的变量能在本类和子类里被使用
  2. 受保护的方法不能在非子类中调用

​

私有变量和方法(private)

​ 双下划线

多态(polymorphism)

​ 定义：实现同一个功能的方法可以有不同的表现形态。

​ note：

  1. 方法同名但参数类型不同，或者同一个方法能适应于不同类型的调式场景。
  2. 不允许两个方法同名但参数个数不同。（java是允许的）

import && module

​ modules:

  1. 自定义模块
  2. 内置标准模块
  3. 开源模块 PyPI · The Python Package Index

导入现有模块

​ 使用方式

  1. import moduleName as elseName

    e.g import numpy as np

  2. from moduleName import className(methodName/propertyName)

    e.g from numpy import sin

导入包中模块

​ 如果以模块的形式复用代码出现了模块冲突的情况，则无法导入实现功能不同但是名字相同的模块，为了解决这个问题，可以用包的形式来复用现有功能。

可以导入整个模块也可以精确导入某个模块的某个方法，速度和范围的差别。

isinstance() && type()

用法：

isinstance(object,classinfo)

用法：

type(object)

isinstance() 与 type() 区别：

  • type() 不会认为子类是一种父类类型，不考虑继承关系。

  • isinstance() 会认为子类是一种父类类型，考虑继承关系。

如果要判断两个类型是否相同推荐使用 isinstance()。

6月8号 :sunny:

异常(exception)

​ 处理原则:出现异常不要紧，但要把异常影响范围限制到最小。

​ 实现要点：

  1. 正确的提示异常信息。
  2. 合理监控和设置监控范围和异常处理的措施。
  3. 使用finally从句回收系统资源。

常用的异常处理类（被动）

异常处理类名 触发场景
IndexError 索引出错触发
ValueError 类型参数正确，但返回值不符合预期是会触发
TypeError 函数参数类型不正确时会触发
ZeroDivisionError 除零出错
AttributeError 引用属性错误触发
OSError 无法完成操作系统级的任务时，会触发该类异常。(无法打开文件)
FloatingPointError 浮点类计算错误
OverflowError 数值运算时超过此种类数值的最大范围
UnicodeTranslateErrror Unicode转换时出错

Raise（主动）

​ 自定义的异常描述信息会更具有操作性。

Finally

Python 垃圾回收机制，只能帮我们回收变量、类对象占用的内存，而无法自动完成类似关闭文件、数据库连接等这些的工作。

​ 作用：资源回收(比如关闭文件、数据库连接等这些的工作)

​ 功能：无论try块是否发生异常，最终都要进入finally语句，并执行其中的代码块。

参考链接：Python try except finally：资源回收 (biancheng.net)

参数值 含义
r 读取模式
w 写入模式。若文件不存在，则创建一个文件，写入；若文件存在，则清空再写入。
r+ 读写模式。从文件头开始写，保留原文件中没有被覆盖的内容。
w+ 读写模式。若文件不存在，则创建一个文件，写入；若文件存在，则清空再写入。
a 附加写模式(不可读)。若文件不存在，则创建一个文件；若文件存在，写入的数据会被加到文件末尾，即文件原来的内容会被保留。
