deploy_freshrss_by_docker

​

​ 因为freshrss的颜值更高，所以选择了freshrss作为rss的阅读器。

​

​ 以普通用户账户进行部署freshrss

部署

# 创建freshrss目录
mkdir freshrss

cd freshrss

vim docker-compose.yml
# docker-compose.yml里的内容

version: "3"

services:
  freshrss-db:
    image: postgres:latest
    container_name: freshrss-db
    hostname: freshrss-db
    restart: unless-stopped
    volumes:
      - freshrss-db:/var/lib/postgresql/data
    environment:		# 数据库默认配置，建议修改
      POSTGRES_USER: freshrss
      POSTGRES_PASSWORD: freshrss
      POSTGRES_DB: freshrss

  freshrss-app:
    image: freshrss/freshrss:latest
    container_name: freshrss-app
    hostname: freshrss-app
    restart: unless-stopped
    ports:
      - "8080:80"	# 建议修改8080端口，注意若防火墙开启，要记得将端口打开
    depends_on:
      - freshrss-db
    volumes:
      - ./data:/var/www/FreshRSS/data
      - ./extensions:/var/www/FreshRSS/extensions
    environment:
      CRON_MIN: '*/45' # rss的刷新周期，单位为分钟，此为45分钟刷新一次
      TZ: Asia/Shanghai  # 时区

volumes:
  freshrss-db:

复制后最好将中文注释给删除。

:wq保存后退出。

docker-compose up -d # 后台运行

freshrss配置

​ 访问ip:port

数据库的配置

数据库配置 上表的信息
主机 是docker中该容器的ip地址，下方会告诉怎么操作获得ip地址
用户名 配置文件中的POSTGRES_USER freshrss
密码 配置文件中的POSTGRES_PASSWORD freshrss
数据库 配置文件中的POSTGRES_DB freshrss
表前缀 随意填

OK，讲一下主机的ip地址的获取方式。

首先用的是postgres数据库，所以查的是postgres数据库，而不是freshrss的ip地址。

# 获取正在运行的容器的一些信息，主要目的是为了获取postgres的id
docker ps

# 获取指定容器的详细信息
docker inspect <container id>

以上即为主机地址。
