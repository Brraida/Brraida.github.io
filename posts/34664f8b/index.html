搭建云盘

    看到大佬发的一个云盘，看起来很nice，心动就行动。其实搭建云盘还有一个目的就是为了收集材料方便。以前尝试在校园网在windows上搭建ftp服务器，在收集材料方面上确实方便了不少。但是因为中文字符，有些材料收集不上来。其实这个搭建得还是比较麻烦的，完全可以去找一些步骤更简单的云盘来搭建。

分享一个漂亮的云盘程序 vue-file-manager - 思有云 - IOIOX

    说个前提，大佬发的是版本是1.8.3.13的。若是按官网教程安装，会出一堆php的deprecated。其原因是php版本太高了，安装php7.4即可。

    以下是我经过测试搭建的教程，有些有些仍有问题，但是总体功能用起来没有问题。主要的是他能很好的在服务器上跑起来了：）

安装nginx和php

sudo apt update && sudo apt upgrade -y

# 安装nginx
sudo apt install nginx -y

# 检测nginx的状态，可以按ctrl+c退出
systemctl status nginx

# 安装必须的依赖
sudo apt-get install ca-certificates apt-transport-https software-properties-common wget curl lsb-release -y

# 导入并安装GPG密钥和存储库
curl -sSL https://packages.sury.org/php/README.txt | sudo bash -x

# 刷新APT存储库列表
sudo apt update

# 更新软件包
sudo apt upgrade -y

# 安装php7.4与其插件
sudo apt install php7.4 php7.4-curl php7.4-common php7.4-cli php7.4-mysql php7.4-mbstring php7.4-fpm php7.4-xml php7.4-zip php7.4-bcmath  php7.4-sqlite3 php7.4-gd php7.4-intl php7.4-exif -y

安装MariaDB

sudo apt install mariadb-server -y

# 检查版本号
mysql -V

# 会让你设置数据库的密码和其他一些列进阶的操作。看自己的需求，默认选是即可。
mysql_secure_installation

# 登录数据库
mysql -u root -p

# 创建数据库
CREATE DATABASE prod_vuefilemanager;

# 授权，其中mysecret是数据库的密码
GRANT ALL ON prod_vuefilemanager.* to 'prod_vuefilemanager'@'localhost' IDENTIFIED BY 'mysecret';

# 刷新权限
FLUSH PRIVILEGES;

# 退出
quit;

    以上部分在到不会与程序运行相关，只有到程序配置时才会用到，所以在页面未出现时，只可能是nginx和php配置出现问题。与数据库这部分不会有相关。

部署与配置

将云盘程序文件上传到/var/www文件夹中，并将文件重命名为vueFileManager。

# 给文件权限
sudo chown -R www-data:www-data /var/www/vueFileManager/storage &&
sudo chmod -R 755 /var/www/vueFileManager/storage &&
sudo chown -R www-data:www-data /var/www/vueFileManager/bootstrap/cache &&
sudo chmod -R 755 /var/www/vueFileManager/bootstrap/cache &&
sudo chown -R www-data:www-data /var/www/vueFileManager/storage &&
sudo chmod -R 755 /var/www/vueFileManager/.env

### 以上给文件755权限后，在运行程序后，还是出现了权限不足的问题，
### 因此给了一个777的权限
sudo chmod -R ugo+rw /var/www/vueFileManager/storage
sudo chmod -R ugo+rw /var/www/vueFileManager/.env

### 刚开始以为是用户的问题，所以先改将文件的用户和用户组改回了root
### 个人觉得不改也没关系，因为nginx的用户中就有www-data
sudo chown -R root:root /var/www/vueFileManager/storage
sudo chown -R root:root /var/www/vueFileManager/.env



# 修改php配置
nano /etc/php/7.4/fpm/php.ini

# 修改的部分
memory_limit = 256M
upload_max_filesize = 128M
post_max_size = 128M
max_file_uploads = 50
max_execution_time = 3600

# 服务重启
service php7.4-fpm reload

# 配置nginx，位置在/etc/nginx/nginx.conf。在http部分最底下添加以下 client_max_body_size 1024M;
http {
    ...
    client_max_body_size 1024M;
}

# 重启nginx
systemctl reload nginx



# 创建vuefilemanager的nginx配置文件
nano /etc/nginx/sites-available/vuefilemanager.conf

# 配置信息
server {
    listen 80;
    listen [::]:80;
    # Log files for Debugging
    access_log /var/log/nginx/laravel-access.log;
    error_log /var/log/nginx/laravel-error.log; 
# Webroot Directory for Laravel project
    root /var/www/vueFileManager/public;
    index index.php index.html index.htm;
    # Your Domain Name
    server_name example.com;
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
# PHP-FPM Configuration Nginx
    location ~ \.php$ {
        try_files $uri =404;
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        fastcgi_pass unix:/run/php/php7.4-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}

    记得修改成自己的域名。如果没有域名，想要访问ip就访问到该云盘程序，那么就得把默认配置文件删除。用该配置文件进行代替即可。

# 软连接
ln -s /etc/nginx/sites-available/vuefilemanager.conf /etc/nginx/sites-enabled/

# 测试配置文件
nginx -t

# 重启配置nginx
systemctl restart nginx

创建定时任务

crontab -e

# 添加以下信息
* * * * *  cd /var/www/vueFileManager&& php artisan schedule:run >> /dev/null 2>&1

后面就会有指导程序界面了，可能会需要用到以下一些资料与命令

SHOW GLOBAL VARIABLES LIKE 'PORT';

# 在数据库终端中输入以下命令，获得数据库的用户和host
select user, host from mysql.user;
